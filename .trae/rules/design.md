# 开发规范与偏好

## 1. 技术栈强制要求
- **组件库**：必须优先使用 **TDesign** 组件库。
- **样式工具**：布局与非主题样式必须使用 **UnoCSS**。
- **颜色系统**：所有颜色必须引用 **TDesign 的设计令牌 (Design Tokens)** (如 `var(--td-brand-color)`)，严禁硬编码 Hex/RGB 值，以确保主题切换能力。

## 2. 设计风格：拟物主义 (Skeuomorphism)
- **核心视觉**：界面元素需呈现真实的物理质感（如塑料、玻璃、金属或纸张感）。
- **实现手段**：
  - **阴影与高光**：利用 UnoCSS 的工具类（如 `shadow-2xl`, `shadow-inner`）或自定义 CSS，为按钮、卡片添加多层阴影和高光，营造立体凸起或凹陷效果。
  - **渐变与纹理**：在 TDesign 组件的背景上叠加细腻的线性渐变或噪点纹理，避免纯色扁平块。
  - **边框与圆角**：使用高光和阴影模拟物理边缘，圆角需自然柔和。
  - **交互反馈**：点击时需有明显的物理按压感（通过 `active:scale-95` 或阴影变化实现）。
- **冲突处理**：当 TDesign 默认样式过于扁平时，**必须**通过 UnoCSS 类名或 `style` 属性覆盖默认样式，以符合拟物风格要求。

## 3. 代码生成原则
- 生成代码时，自动组合 TDesign 组件结构 + UnoCSS 拟物样式类。
- 确保代码可维护性，尽量将复杂的拟物样式提取为 UnoCSS 快捷规则或 CSS 变量。
