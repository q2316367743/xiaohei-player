### 🧠 产品名称构想（暂定）
**Mirror (镜像)** 或 **Memora (记忆伙伴)**
*Slogan：记录，是为了让另一个自己记得。*
---
### ⚙️ 核心架构（技术栈：Tauri + Rust + VecDB + LLM）
#### 1. 存储层
*   **Memo DB (SQLite/PostgreSQL):** 存储原始笔记、结构化数据（待办、纪念日）、评分权重。
*   **Vector DB (如 SQLite-VSS 或 Qdrant):** 存储文本向量化后的 Embeddings。
    *   *来源：* 用户主动撰写的 Memo + AI 从聊天中提取的精华。
#### 2. 引擎层
*   **清洗与沉淀 Daemon:**
    *   **输入：** 聊天记录（标记为 `pending`）。
    *   **触发：** App 启动时 / 后台空闲时。
    *   **逻辑：** 批量处理 -> LLM 提取关键信息 -> 生成摘要 -> 存入 Memo DB & 向量化。
    *   **输出：** 一条带时间戳的“记忆沉淀记录”。
#### 3. 交互层
*   **输入端：** 极简输入框（支持语音/文本）。
*   **记忆端（卡片流）：** 基于“权重算法”和“时间衰减”的智能展示，而非死板的时间列表。
*   **社交端（朋友圈）：** AI 每日/定期发布的动态总结。
---
### ✨ 四大核心功能亮点
#### 1. 🤖 AI 好友（带记忆的第二人格）
*   **特征：** 多角色设定（知心姐姐、毒舌教练、疯狂科学家）。
*   **能力：** 聊天时实时检索向量库，能“记得”你三年前的梦想、上周的痛苦、刚才说的废话。
*   **价值：** 解决 AI “失忆”和“冷漠”的问题，提供情绪价值和深度对话。
#### 2. 🔄 自动记忆沉淀（策略 B 完美闭环）
*   **痛点解决：** 用户懒得打字整理，只想闲聊。
*   **机制：** 闲聊 -> 后台静默提取 -> 自动转为 Memo。
*   **体验：** “你说我听，你忘我记”。每次启动 App，AI 都会汇报：“我不在这段时间，帮你整理了这几条记忆。”
#### 3. 📱 AI 朋友圈（时光机）
*   **玩法：** AI 代替用户发朋友圈。
*   **内容：** “历史上的今天”、跨时空的对比（当年的傻事 vs 现在的成熟）、矛盾的 Flag。
*   **价值：** 用社交化的方式解决“复习”枯燥的问题，让用户忍不住去翻看旧笔记。
#### 4. 💡 神评论与记忆唤醒
*   **即时反馈：** 用户写 Memo，AI 基于过去记忆秒回神评。
*   **长期唤醒：** 随机漫步功能，随机弹窗一条旧 Memo，刺激大脑回忆。
---
### 🚀 开发优先级建议 (MVP 路径)
为了能尽快上线验证想法，建议分三步走：
**Phase 1: 核心记忆闭环 (MVP)**
1.  **基础 Memo:** 记录、简单的标签管理。
2.  **基础聊天:** 接入 LLM API，实现对话。
3.  **向量存储:** 实现 Memo 的向量化存储和检索（RAG）。
4.  **基础沉淀:** 实现聊天记录 -> LLM 提取 -> 存为 Memo 的自动化流程。
    **Phase 2: 情感与交互体验**
1.  **角色系统:** 加入 Prompt 模板，让 AI 有性格。
2.  **朋友圈功能:** 实现每日定时推送 AI 总结的朋友圈。
3.  **评论互动:** AI 对 Memo 的自动评论。
    **Phase 3: 深度玩法与仪式感**
1.  **策略 B 优化:** Tauri 后台静默处理的进度条和通知。
2.  **权重算法:** 实现基于用户行为的笔记权重衰减。
3.  **多模态:** 图片 Memo 的支持与 AI 配图生成。
